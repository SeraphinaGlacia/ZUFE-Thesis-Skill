## 一、LaTex 确实强大，但却罕为人知

在本科阶段，论文、专业实践报告和课程小论文是非常常见的写作任务。对于学生而言，真正困难的地方往往不只是内容写作本身，还有大量繁琐、重复、容易出错的格式调整工作：封面、目录、页眉页脚、标题层级、图表编号、参考文献格式、摘要排版等。这些工作消耗了学生大量的时间，却并不直接提升论文内容质量。

LaTeX 本来是解决这些问题的优秀工具。它能够将内容和格式分离，自动生成目录、编号、引用和参考文献，使论文排版更加规范、稳定、可维护。然而事实上，在现实中，很多学生并不愿意使用 LaTeX。

## 二、事实上， 先进工具并不一定会被广泛使用

我认为：一个工具是否值得学习，并不只取决于它是否先进，还取决于它的学习成本能否被未来的多次使用摊匀。

Markdown 就是一个很典型的例子：它的学习门槛低，使用场景多，可以用于笔记、知识库、项目文档和 AI 交互，因此一旦掌握，就会在之后的学习和工作中不断复用，形成明显的复利。

但 LaTeX 对很多学生来说却不一样。它当然强大、规范、专业。但初次门槛较高，需要安装环境、理解模板、学习语法、处理编译，要面对陌生的报错信息。

对于很多大学生来说，整个本科阶段可能只需要完成三到四次正式论文或实践报告写作。对于这些学生而言，专门学习 LaTeX 很可能变成一次“高成本、低收益”的投入。

LaTex 本身是好的，但学习它的效费比并不总是足够高。相比于投入较多的精力学习一项回报未知的技能，大多数学生还是会选择继续使用 Word。

## 三、我们让 LaTeX 的好处先发生

> [!NOTE]
> 一个完全不会 LaTeX、也不懂论文模板结构的人，只要上传 Word、回答几个简单问题，就能得到规范 PDF。这是项目目标，因为 skill 需要做到把复杂度封装起来，让不懂 LaTeX 的人也能享受 LaTeX 的排版能力。

因此，考虑到现实情况，这个 skill 的设计初衷不是强迫学生掌握 LaTeX，而是降低接触 LaTeX 的心理门槛，即：**先通过 skill 享受 LaTeX 的好处，再决定要不要学习 LaTeX**。

学生仍可以按照原有习惯使用 Word 甚至是普通文本完成初稿，而 skill 则负责识别文档结构、整理章节层级、填入学校论文模板、处理图表和参考文献，并最终生成规范的 LaTeX 工程与 PDF 文件。

在这种情况下，LaTeX 就不再是学生必须首先掌握的新工具。学生只需要提供 Word 格式的文章内容，skill 负责完成规范化和自动编译（甚至是傻瓜式的引导式问答）。最终成果既可以是可直接提交的标准 PDF，也可以是完整的 .tex 工程。

我认为这样具有两层价值：

1. 对于只想完成一次专业实践或毕业论文的学生来说，或许他们确实对 LaTex 不感兴趣，但 skill 配合模版可以显著减少无意义的精力浪费，让他们也能体会到 Latex 的好处；

2. 而对于愿意进一步了解 LaTeX 的学生来说，它又可以作为一个温和的入门入口：学生可以通过观察生成的 LaTeX 工程，理解论文结构如何被组织，参考文献如何被管理，图表、目录又是如何自动配置的。

所以，这个 skill 想解决的是：**如何让学生在不懂 LaTeX 的情况下，也能先体验到 LaTeX 的好处。当好处先产生，学习才可能从负担变成兴趣。**

ZUFE Thesis Skill 的目标，是把不规范的 Word 转化为规范的 LaTeX 工程和 PDF 成果；把论文排版从手工调格式，变成自动化、结构化、可复用的流程；也让 LaTeX 从一个让人害怕的新事物，变成学生能先享受成果，后继续学习的强大工具。
