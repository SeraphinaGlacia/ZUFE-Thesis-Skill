# ZUFE-Thesis-Skill 常见问题

## 这是什么项目？

ZUFE-Thesis-Skill 是一个配合 ZUFE-Thesis LaTeX 模板使用的非官方 Agent Skill。它帮助用户把已经写好的 Word 初稿转换为 LaTeX 工程，并进一步生成 PDF。

它可以被视为是一种自动排版辅助工具。

它主要面向不熟悉 LaTeX、但希望使用 ZUFE-Thesis 模板完成论文或报告排版的用户。

## 这是浙江财经大学官方工具吗？

不是。

本项目不代表浙江财经大学官方工具，也不是原始 ZUFE-Thesis 模板项目的官方组成部分。它只是一个辅助 Skill，用来配合原始模板降低使用门槛。最终格式仍应以学校要求、导师要求和原始模板为准。

## 它和 ZUFE-Thesis 模板是什么关系？

ZUFE-Thesis 是原始 LaTeX 模板，负责提供论文工程结构和格式规则。

ZUFE-Thesis-Skill 是围绕这个模板制作的辅助 Skill，负责引导 Agent 检查环境、读取 Word、整理中间结构、写入模板、编译 PDF，并在不确定时向用户确认。

简单说：

- ZUFE-Thesis 是模板。
- ZUFE-Thesis-Skill 是帮助普通用户使用模板的 AI 辅助层。

## 我不会 LaTeX，可以用吗？

可以，这正是本项目的主要目标之一。

用户仍然可以用 Word 写初稿，然后让 Agent 借助本 Skill 把内容写入 LaTeX 模板。你不需要先系统学习 LaTeX，但需要会基本操作 Agent 工具，并愿意检查最终生成的 PDF。

## 它能不能把 Word 一键变成可以直接提交的最终 PDF？

不应该这样理解。

本 Skill 会尽量自动完成模板检查、内容转换、编译和质检，但 Word 结构可能不清晰，AI 也可能需要确认章节、摘要、关键词、图表标题、参考文献等内容。最终 PDF 仍然必须由用户自己检查，不建议未经检查直接提交。

## 我需要准备什么？

通常需要准备三类东西：

- 一份已经写好的 `.docx` Word 初稿。
- 一份完整的 ZUFE-Thesis 模板项目。
- 一个可以运行 Agent Skill 的工具环境，例如 Kimi Work、Codex 或类似工具。

本 Skill 需要和完整的模板项目放在同一个工作区中使用。

## 这个 Skill 怎么安装？

通常推荐通过项目 README 中的 `npx skills add SeraphinaGlacia/ZUFE-Thesis-Skill` 安装。

如果无法使用 `npx` 或网络环境不稳定，可以查看项目 release 中是否提供 ZIP 包，并按 README 的说明手动安装。无论采用哪种方式，最终都应保证 `zufe-thesis-typesetter/` 这个 Skill 包被安装到 Agent 能识别的位置。

## 使用后会得到哪些文件？

一次完整转换后，通常会得到：

- `main.pdf`：按 ZUFE-Thesis 模板编译出的 PDF。
- 可继续维护的 LaTeX 工程文件。
- `report.md`：转换过程和确认事项说明。
- `qa_report.md`：编译后质量检查结果。

具体文件位置以实际工作区和 Agent 执行结果为准。

## 它会修改我的原始 Word 吗？

本项目的设计目标不是直接改坏用户原始 Word，而是读取 Word 内容并生成 LaTeX 工程。实际使用前仍然建议备份原始 `.docx` 和其他重要文件，避免误操作、覆盖或环境问题造成损失。

## Word 文档写得很乱也可以吗？

可以尝试，但效果会受到影响。

如果 Word 中标题层级、摘要、关键词、图表说明和参考文献比较清晰，转换会更顺利。如果文档结构很混乱，Agent 可能需要更多确认，也可能需要用户手动补充信息。
