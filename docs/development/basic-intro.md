### 文件结构

- 本项目下，除了 `docs/` 和 `output/` 文件夹外，其他文件和文件夹都是原始 LaTex 模版项目中已经拥有的文件，`docs/` 存放相关的说明文档、开发设计等资料，`output/` 下存放开发出来的 skill 文件夹。

### 基本目标

- 开发一个 skill，用户在当前项目（就是最基本的、除了 `docs/` 和 `output/` 外原始模版仓库下）下安装此技能，即可实现将 word 文档转化为 LaTex 工程，并生成 pdf，具体参考 `docs/others/让 LaTeX 的好处先发生：我的 ZUFE Thesis Skill 设计初衷.md` 所述。用户可以在对话框中进行对话交流，技能会自动识别文档内容进行转换。

### 开发抓手

- `docs/development/workflow-design.md`：当前阶段的工作流总览文档。
- `docs/development/workflows/`：A/B/C 流程细节文档，先逐步确认用户侧工作流，再决定项目文件组织。
- `docs/others/让 LaTeX 的好处先发生：我的 ZUFE Thesis Skill 设计初衷.md`：项目愿景与设计初衷。
