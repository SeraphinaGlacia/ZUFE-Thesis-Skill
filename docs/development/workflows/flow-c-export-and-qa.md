# 流程 C：编译、质检与导出

流程 C 在流程 B 生成 LaTeX 工程后启动，负责编译、错误诊断、质量检查和最终交付。

## 当前预览

- 运行 `xelatex -> biber -> xelatex -> xelatex` 编译链。
- 收集日志并诊断错误。
- 生成 `workspace/output/main.pdf`。
- 生成 `workspace/output/report.md` 和 `workspace/output/qa_report.md`。
- 向用户说明最终文件位置、已知风险和需要人工确认的事项。

## 待确认问题

- 编译失败时哪些错误允许自动修复，哪些必须询问用户。
- 质量检查第一版是否只检查日志和文件存在，还是要读取 PDF 页面截图。
- 是否检查模板蓝字、占位符、目录页、摘要页、参考文献页。
- 输出目录中的工程、PDF、报告如何命名。
- 面向非技术用户的最终交付说明应该包含哪些内容。
